Angular.js commit 规范

commit message的格式

包括三个部分：header、body（可选）和footer（可选）

Header

type(必选)

commit type的类别，只允许使用下面7个标识:

  • feat：新功能（feature）
  • fix：修补bug
  • docs：文档（documentation）
  • style： 格式（不影响代码运行的变动）
  • refactor：重构（即不是新增功能，也不是修改bug的代码变动）
  • test：增加测试
  • chore：构建过程或辅助工具的变动

scope（可选）

用于说明 commit 影响的范围，比如数据层、控制层、视图层等等：
例（Angular）：$location, $browser, $compile, $rootScope, ngHref, ngClick, ngView, *（影响不止一个）等。

subject(必选)

简短描述，不超过50个字符
