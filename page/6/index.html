GIT

git常用命令合集


命令 用途备注
git init 初始化仓库
git config --global user.name [用户名] 配置用户名
git config --global user.email [邮箱] 配置邮箱
git config user.name 查看用户名
git config user.email 查看邮箱
git clone [仓库地址] 克隆到本地
git remote -v 查看远程仓库
git remote add [分支名称] [仓库地址] 创建并关联本地到远程仓库
git remote rm origin 删除远程仓库
git pull [分支名称] 同步分支
git merge [分支名称] 合并分支
git fetch [分支名称] 不更新本地分支，另需merge
git reflog 查看commit版本号
git reset --hard HEAD 回退到上个版本
git reset --hard [版本号] 回退到某个版本
git push -f origin master 强制提交
git branch 查看本地分支
git checkout -b [分支名称] 创建新分支并切换到新分支
git branch -D [分支名称] 强制删除分支
git push origin --delete [分支名称] 删除远程分支
git add . 变更内容全部提交到暂存区
commit -m [备注] 提交备注
git -u origin master [分支名称] 第一次本地推送到远程分支
git push origin [分支名称] 本地推送到远程分支
git log 查看commit日志
git rm [文件名] 删除某文件
git status 查看当前版本是否修改
git diff 查看暂存区和工作区的差异
git rm -r --cached . 清除缓存（.gitignore配置失效）

建议：
1.在本地修改与远程代码无冲突的情况下，优先使用：pull->commit->push
2.在本地修改与远程代码有冲突的情况下，优先使用：commit->pull->push
